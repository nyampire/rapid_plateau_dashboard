# ダッシュボード UI 設計書（カバレッジ率中心）

ステータス: **設計レビュー中**（モック実装の前段）
最終更新: 2026-05-23

DESIGN.md §7（フロントエンド）を、**建物カバレッジ率を主眼**とする方針で具体化・差し替えるもの。
地図は主役にせず端の補助要素とする。

---

## 1. UI 方針（確定事項）

- **中心指標 = 建物カバレッジ率（建物加重）**: `Σ(OSMに存在するPLATEAU建物) / Σ(全PLATEAU建物) × 100`。
  「PLATEAU データが OSM でどれだけ表現済みか」を一目で示す。これを最上部の主役 KPI に据える。
- **構造 = 地方別サマリ + 市区町村一覧の2段**。全体像（地方別）→ 個別（市区町村）の順で視線が流れる。
- **地図は補助**。画面端に小さく置き、coverage ポリゴンを率で色分け。クリックで都市詳細を開く程度（主役ではない）。
- 静的 HTML + JS（既存 nginx 配信に `/dashboard/`）。グラフ Chart.js、補助地図 Leaflet。データは §6 API を読むだけ。

### 指標の定義と分母スコープ（要確認の前提）

- **建物カバレッジ率の分母**: 自前 DB 取込済（in_local_db）都市の PLATEAU 建物（outline のみ、building_part 除外）。
  - 取込済だが OSM 未計測の都市は「未計測」として分子・分母から除外（率を歪めない）。計測済み都市での率として明示する。
  - **将来拡張**: 全国 306 都市の総棟数は attributedata の棟数列（例「3施設 9棟」）から取得でき、「全国建物総数に対する真のカバレッジ」に拡張可能。初期は計測済み都市ベースで表示し、注記する。
- **補助 KPI（都市加重）**: 都市カバレッジ = 取込済 / 306、wiki 完了 = done / 306。母集団 306 は全国 PLATEAU 提供都市。

## 3. コンポーネント仕様とデータ束ね

| # | コンポーネント | 表示内容 | データ源（API / dash_*） |
|---|---|---|---|
| A | ヒーロー: 建物カバレッジ率 | 大きな % + 前回比(▲▼) + 計測済都市数注記 | `/summary`: dash_progress_history 最新 `overall_rate`, 前回行との差分 |
| B | 補助KPI: 都市カバレッジ | `cities_in_db / cities_total` と % | `/summary`: `cities_in_db`, `cities_total` |
| C | 補助KPI: OSM完了 | `cities_osm_done / cities_total` と % | `/summary`: `cities_osm_done`, `cities_total` |
| D | トレンド スパークライン | overall_rate の週次推移 | `/summary`: dash_progress_history 時系列 |
| E | 地方別サマリカード(8) | 地方名・建物率(色)・取込/全都市数 | `/regions`: dash_city_stats JOIN dash_city_master を region で集計 |
| F | 市区町村一覧 | 都市/地方/建物率バー/PLATEAU棟数/OSM数/wiki状態/LOD | `/cities`: dash_city_master + dash_city_stats |
| G | 補助地図 | coverage ポリゴンを率で色分け、click→詳細 | `/cities.geojson`: plateau_coverage + import_rate |
| H | 都市詳細 | 率内訳・交差数・wiki日付・リンク | `/cities/{code}` |

### 率バー・色分けの凡例（共通）

| 区分 | 建物カバレッジ率 | 色 |
|---|---|---|
| 高 | 75–100% | 緑 |
| 中 | 50–75% | 黄 |
| 低 | 25–50% | 橙 |
| 最低 | 0–25% | 赤 |
| 未計測 | (取込済・OSM未ロード) | グレー（バーなし「未計測」） |
| 未取込 | (not_in_db) | 薄グレー（「未取込」） |

wiki 状態バッジ: `完了`（+`✓検証`）/ `作業中` / `未着手`。

## 4. 状態とデータギャップの扱い（現状データに即して）

- **率は現状 6/144 都市のみ計測済み**。一覧では計測済みを率順上位に、未計測は「未計測」で別扱い。全国一括投入後に埋まる。
- **未取込 162 都市**: PLATEAU 建物が自前 DB に無く率を出せない。一覧にはグレーで掲載（母集団 306 を示すため）、地図には coverage ポリゴンが無いので**描画しない**（地図が補助な理由のひとつ）。
- **トレンドは現状 1 スナップショット**。週次バッチで蓄積。モックではサンプル系列で見せ、注記。
- **boundary_geom(N03) 未取込**: 地図は coverage 凸包を使用。将来 N03 で全 306 都市の正確境界に差し替え（§9-1）。それまで地図は取込済 144 都市のみ。

## 5. インタラクション

- 地方カード クリック → 一覧をその地方にフィルタ + スクロール。
- 一覧ヘッダ クリック → ソート（建物率/棟数/都市名/状態）。
- フィルタ: 地方、状態（取込/未取込・wiki done/in_progress/not_started）、フリーワード（都市名）。
- 一覧行 / 地図ポリゴン クリック → 都市詳細パネル。
- 詳細から OSM / Rapid エディタへの外部リンク（該当 bbox）。

## 6. 必要な API（DESIGN.md §6 を本 UI 向けに具体化）

| エンドポイント | 返却 | 備考 |
|---|---|---|
| `GET /api/dashboard/summary` | 建物カバレッジ率(最新+前回比)、cities_total/in_db/osm_done、トレンド系列 | dash_progress_history |
| `GET /api/dashboard/regions` | 地方別: building-weighted 率、取込/全都市数、done 数 | dash_city_stats ⨝ dash_city_master を region 集計（オンザフライ可、144行で軽量。新テーブル不要）|
| `GET /api/dashboard/cities` | 全 306 都市: master 全項目 + stats(率/棟数) + 状態 | フィルタ(地方/状態) クエリ対応 |
| `GET /api/dashboard/cities/{code}` | 単一都市詳細 | |
| `GET /api/dashboard/cities.geojson` | coverage ポリゴン + import_rate（144都市） | 補助地図用 |

> 地方ロールアップは集計テーブルを増やさず API でオンザフライ集計（dash_city_stats は最大144行）。重ければ将来 MatView 化。

## 7. 技術・実装メモ（モック→本実装）

- **モック段階**: 静的 HTML + ダミー JSON（上記 API 形に一致）で見た目を固める。現状 6 都市 + サンプルでレイアウト検証。
- 本実装: Chart.js（トレンド/率バーは CSS でも可）、Leaflet（補助地図のみ）。同一オリジン配信で CORS 不要。
- レスポンシブ: 主役は表とKPIなので 1 カラム化が容易。地図は補助なのでモバイルでは折りたたみ可。
- アクセシビリティ: 率は色だけでなく数値も併記（色覚配慮）。
